# EventHub — Design Context

> Read this before any UI work. It defines who EventHub is for and how it must feel.
> Skills to run on UI work: `/impeccable`, `/ui-ux-pro-max`, `/polish`.

## Design Context

### Users
General public in a city looking to **discover and join local events** — concerts,
workshops, meetups, community gatherings. They arrive curious but unconvinced, often
on a phone, scanning quickly. The interface must earn trust fast (this is a real
event, real people are going) while feeling alive enough to make joining exciting.
Secondary users are **organizers** who create events and want them to look great.

### Brand Personality
**Energetic · Social · Bold.** EventHub should feel like the buzz before something
worth showing up to (a gig, a workshop, a meetup, a Saturday-afternoon thing) — not
like admin software, and not like a nightlife-only app. Confident, lively, a little
loud, but never cluttered or untrustworthy. Voice is warm and direct: "Find your next
experience," not "Browse our event catalog." Keep copy event-type-agnostic; "night
out" pigeonholes it as clubbing.

Emotional goals: excitement, belonging, momentum. The user should feel *pulled in*.

### Aesthetic Direction
- **Theme:** Light-led (daytime browsing/discovery), with a fully polished dark mode.
- **Editorial, not templated.** Big confident display type, asymmetric layouts,
  varied spacing rhythm, intentional moments — closer to a culture/events magazine
  than a SaaS dashboard.
- **Color:** Electric, confident **blue** as the dominant brand color, with a single
  **warm accent** (coral/tangerine) used sparingly (≈10%) for energy and CTAs.
  This warm+cool pairing is what makes it read "social/energetic" instead of
  "corporate blue." Neutrals are tinted slightly toward the blue hue for cohesion.
- **Type pairing:** Display = **Bricolage Grotesque** (characterful, contemporary,
  energetic). Body = **Hanken Grotesk** (clean, warm, highly readable). Both via
  Google Fonts. No Inter / Roboto / system-default monoculture.

### Anti-References (do NOT look like)
The **generic AI dashboard**. Explicitly avoid:
- Identical icon-card grids (icon + heading + text, repeated) — the current Home
  page's biggest tell. Vary card sizes, layouts, and content density.
- Purple→blue gradients, cyan-on-dark, neon-on-dark, glassmorphism everywhere.
- Gradient text, thick colored left-border stripes on cards/alerts.
- Everything centered, everything evenly spaced, everything in a card.

### Design Principles
1. **Earn the energy.** Bold display type + one warm accent + motion on key moments.
   Energy comes from contrast and rhythm, not from coloring everything in.
2. **Hierarchy through restraint.** Mute secondary text (don't set body copy to full
   contrast), vary weight and size sharply, keep one primary action per view.
3. **Editorial over templated.** Break the symmetric grid. Asymmetry, varied spacing,
   and real content beat the icon-card matrix.
4. **Mobile is the main stage.** Most users are on a phone — adapt layouts, don't
   shrink them. Generous touch targets, no amputated features.
5. **Trust is a feature.** Real dates, real locations, real attendee signals shown
   clearly. Excitement without ever feeling like a spammy ticketing site.

### Technical Constraints
- React 19 + Vite + Tailwind CSS 3. Framer Motion available for motion.
- Dark mode toggles a `dark` class on `<html>` (some legacy pages also thread a
  `darkMode` prop — migrate toward CSS-variable tokens over time).
- Keep scope pragmatic: this is a final-year project, not enterprise. Favor
  high-impact, low-churn changes; reuse the shared token layer in `src/index.css`.
